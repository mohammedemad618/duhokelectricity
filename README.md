# الموقع الرسمي لدائرة توزيع كهرباء دهوك

موقع إلكتروني رسمي حديث لدائرة توزيع كهرباء دهوك وفروعها: واجهة عامة للمواطنين
(أخبار، إعلانات، فروع، أقسام، شكاوى) + **لوحة تحكم داخلية للموظفين** مع نظام
تسجيل دخول وصلاحيات، كلّها مدعومة بقاعدة **MongoDB**.

## ✅ المُنفَّذ

- **الواجهة العامة** (المرحلة 1): الرئيسية، الأخبار (+فلاتر)، تفاصيل الخبر،
  الإعلانات، الفروع، الأقسام، الشكاوى، اتصل بنا، الأسئلة الشائعة — عربية RTL،
  متوافقة مع الهاتف.
- **لوحة التحكم** (المرحلة 2): تسجيل دخول، جلسات JWT، حماية المسارات،
  نظام صلاحيات بخمسة أدوار، إدارة الأخبار (سير مراجعة ونشر)، الإعلانات،
  الشكاوى، والمستخدمين.
- **قاعدة البيانات** (المرحلة 3): MongoDB حقيقية لكل المحتوى والشكاوى
  والمستخدمين وسجلّ العمليات (Audit Log).
- **التقارير والإحصائيات** (المرحلة 4): لوحة تقارير تحليلية (الأخبار/الشكاوى
  حسب الحالة والتصنيف والفرع والقسم)، سجلّ العمليات، وتصدير CSV + طباعة/PDF.
- **تعدّد اللغات**: واجهة الموقع العام ولوحة التحكم بثلاث لغات — العربية،
  الكردية، الإنجليزية — مع ضبط الاتجاه (RTL/LTR) تلقائياً ومبدّل لغة.

## 🚀 التشغيل

### أول مرة

```bash
npm install

# 1) شغّل قاعدة البيانات المحلية في نافذة طرفية مستقلّة (اتركها مفتوحة)
npm run db

# 2) في نافذة أخرى: عبّئ البيانات التجريبية والمستخدمين
npm run seed

# 3) شغّل الموقع
npm run dev      # http://localhost:3000
```

### لاحقاً

شغّل `npm run db` (نافذة) و `npm run dev` (نافذة أخرى). البيانات محفوظة على
القرص في `.mongo-data/`.

### للإنتاج

عيّن `MONGODB_URI` في `.env.local` إلى رابط **MongoDB Atlas** (لا حاجة لـ
`npm run db`)، ثم: `npm run build && npm start`.

### النشر على Netlify

المشروع جاهز للنشر على Netlify عبر `netlify.toml`.

إعدادات البناء:

- Build command: `npm run build`
- Publish directory: `.next`
- Node version: `22`

قبل النشر، أضف متغيرات البيئة في Netlify من:
Project configuration → Environment variables

```env
MONGODB_URI=mongodb+srv://...
MONGODB_DB=dohuk_electricity
AUTH_SECRET=ضع_قيمة_عشوائية_طويلة_هنا
```

ملاحظات:

- استخدم MongoDB Atlas في الإنتاج، ولا تشغّل `npm run db` على Netlify.
- لا ترفع `.env.local` إلى Git؛ استخدم لوحة Netlify للمتغيرات السرية.
- `NETLIFY_NEXT_SKEW_PROTECTION=true` مضبوط في `netlify.toml` لتقليل مشاكل
  الانتقال بين الإصدارات أثناء التحديثات.

## 🔐 الأدوار والصلاحيات (القسم 11)

| الدور              | الصلاحيات                                                        |
| ------------------ | --------------------------------------------------------------- |
| **المدير العام**    | كل شيء: المستخدمون، كل الفروع/الأقسام، النشر، الشكاوى، الإعدادات |
| **مدير الفرع**      | محتوى وإعلانات وشكاوى فرعه + مراجعة ونشر محتوى الفرع            |
| **مسؤول القسم**     | إضافة محتوى قسمه + متابعة الشكاوى المُحالة لقسمه                |
| **محرر المحتوى**    | إنشاء/تعديل محتواه فقط + إرساله للمراجعة (لا ينشر)              |
| **المراجِع**        | قبول/رفض ونشر المحتوى + إضافة ملاحظات                          |

**سير نشر المحتوى:** مسودة → قيد المراجعة → منشور (أو مرفوض → إعادة للتعديل) → مؤرشف.

## 🌐 تعدّد اللغات (i18n)

الواجهة العامة ولوحة التحكم تدعمان ثلاث لغات: **العربية** (افتراضي، RTL)، **الكردية**
(RTL، خط عربي)، **الإنجليزية** (LTR). تُختار اللغة عبر مبدّل في الهيدر ويُحفظ
الاختيار في كوكي، ويُضبط اتجاه الصفحة (`dir`) واللغة (`lang`) تلقائياً.

- **ملفات الترجمة**: [src/lib/i18n/dictionaries/](src/lib/i18n/dictionaries/)
  — `ar.ts` هو المرجع، و`en.ts`/`ku.ts` يطابقانه (يفرض TypeScript تطابق المفاتيح).
  لتعديل أي نص، حرّر الملف المناسب فقط.
- **الترجمة الكردية بلهجة بادينية (كرمانجية / خط عربي)** المناسبة لدهوك،
  ويُفضّل **مراجعتها النهائية من مترجم مختص** في الدائرة.
- **النطاق**: واجهة الموقع العام ولوحة التحكم مترجمتان. **محتوى** الأخبار/الإعلانات
  يبقى بلغة كتابته (ترجمة المحتوى نفسه تحتاج حقول لغوية في قاعدة البيانات — خطوة
  لاحقة).
- ملاحظة فنية: قراءة كوكي اللغة في التخطيط الجذري تجعل صفحات الموقع تُعرَض
  ديناميكياً؛ اعتماد مسارات لكل لغة (`/en`, `/ku`) لاحقاً يعيد التوليد الثابت
  ويحسّن SEO.
